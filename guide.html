tips_and_updates Pro Tip

코드가 길수록 분석 품질이 올라갑니다. 특정 함수만 잘라내는 것보다 파일 전체를 붙여넣는 것이 중복·잔재 코드 탐지에 훨씬 효과적입니다.

지원 언어

JS · TS · Python · Go · Java · C# · Swift · Kotlin · Rust · PHP · HTML · CSS 외

lock

내 코드는 안전한가요?

입력한 코드는 분석 후 서버에 저장되지 않습니다. 분석 결과만 브라우저 로컬 스토리지에 보관되며, 이 기기에서만 열람할 수 있습니다.

  • check_circle 코드 원문 서버 전송 없음
  • check_circle 로그인·회원가입 불필요
  • check_circle 최대 50개 기록 자동 보관

02. Diagnostics

결과 해석 — 심각도 3단계

각 이슈 카드를 클릭하면 상세 분석 드로어가 열립니다. "이 코드가 하는 일 / 지금도 필요한가요? / AI 권장 조치" 순서로 확인하세요.

치명적 즉시 처리 권장

Critical

하드코딩된 API 키·비밀번호, 코드베이스 어디서도 호출되지 않는 사용 중단 함수·클래스, 보안 취약점 패턴입니다. 즉시 삭제 또는 수정이 필요합니다.

대표 예시

const API_KEY = "sk-abc123"

function legacyAuth() { ... }

주의 검토 후 처리

Warning

다른 파일에 이미 존재하는 중복 로직, 주석 처리된 채 방치된 구버전 코드 블록, 이전 AI 세션 시도의 잔재 코드입니다. 문맥을 확인 후 통합하거나 삭제하세요.

대표 예시

function formatData() { /* 중복 */ }

/* TODO: 리팩토링 예정 ... */

참고 선택적 적용

Info

성능 저하를 유발하는 패턴, 개선 가능한 의존성 배열, 미사용 import 등입니다. 즉각적인 버그가 아니므로 개발자 판단에 따라 선택적으로 적용합니다.

대표 예시

useEffect(() => {}, [props])

import { unused } from './utils'

03. Detail Panel

상세 분석 드로어

이슈 카드를 클릭하면 오른쪽에서 드로어가 슬라이드되어 열립니다. 단순한 경고를 넘어 각 코드 블록에 대한 맥락을 이해하고 결정하세요.

  • search
    이 코드가 하는 일

    AI가 해당 코드 블록의 역할을 평문으로 설명합니다. 코드를 읽지 않아도 기능을 파악할 수 있습니다.

  • help
    지금도 필요한가요?

    코드베이스 내 호출 여부, 다른 모듈과의 관계 등 실제 필요성 근거를 구체적으로 제시합니다.

  • auto_fix_high
    AI 권장 조치

    삭제·유지·통합 중 하나를 코드 예시와 함께 제안합니다. 최종 결정은 항상 사용자에게 있습니다.

04. History

리뷰 기록 — 로그인 없는 저장

storage
브라우저 로컬 저장소

분석 완료 시 결과(이슈 목록·카운트)가 브라우저의 localStorage에 자동 저장됩니다. 서버·DB·계정이 필요 없습니다.

devices
이 기기에서만 열람

저장된 기록은 분석을 실행한 브라우저에서만 볼 수 있습니다. 다른 기기로 공유하려면 결과 화면을 직접 캡처하세요.

history
최대 50개 자동 보관

오래된 기록부터 자동으로 삭제되어 항상 최근 50개를 유지합니다. 리뷰 기록 페이지에서 개별 삭제도 가능합니다.

자주 묻는 질문

입력한 코드가 서버에 저장되나요?
add

코드 원문은 분석을 위해 Groq API 서버(미국 소재)로 전송되지만, 분석 완료 즉시 폐기됩니다. 로컬 서버에도, Vibe Review 서버에도 코드 원문은 저장되지 않습니다. 분석 결과(이슈 목록)만 여러분의 브라우저에 보관됩니다.

어떤 프로그래밍 언어를 지원하나요?
add

Groq의 Llama 3.3 모델 기반이므로 대부분의 언어를 이해합니다. JavaScript, TypeScript, Python, Go, Java, C#, Swift, Kotlin, Rust, PHP, HTML, CSS 등에서 잘 동작하며, 특히 JavaScript/TypeScript 계열에서 탐지 품질이 높습니다.

분석에 얼마나 걸리나요?
add

코드 크기와 서버 상태에 따라 다르지만 평균 3~8초입니다. Groq는 특히 빠른 추론 속도를 제공하여 100줄 내외의 코드는 3초 이내에 결과가 나옵니다. 코드가 길수록 시간이 늘어나지만 40만 자(약 10,000줄) 이하라면 모두 처리 가능합니다.

백엔드 서버가 없으면 동작하지 않나요?
add

현재는 로컬 Node.js 서버(포트 3001)를 직접 실행해야 합니다. cd server && npm run dev 명령으로 서버를 먼저 시작한 뒤 분석을 진행하세요. 서버가 꺼져 있으면 분석 버튼 클릭 시 에러 배너가 표시됩니다.

AI가 틀린 분석을 할 수도 있나요?
add

네, AI 분석은 완벽하지 않습니다. 특히 여러 파일에 걸친 참조 관계는 붙여넣은 코드만으로는 파악하기 어렵습니다. 드로어의 "무시하기" 버튼으로 오탐을 표시해두고, 최종 삭제 여부는 항상 직접 판단하세요. Vibe Review는 의사결정을 돕는 보조 도구입니다.
